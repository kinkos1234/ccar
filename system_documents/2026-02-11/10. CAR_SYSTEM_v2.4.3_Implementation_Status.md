## 6. 아키텍처 변경 사항 (문서 반영 필요)

### 6.1 내부 연락처 시스템 (신규)

```
User ──< CarInternalContact >── CAR
        (carId + userId 복합 PK)
```

- 프론트엔드: `InternalContactSelector` 컴포넌트 (타입어헤드 다중 선택)
- 백엔드: CAR 생성/수정 시 `internalContactUserIds[]` 배열로 관리
- API: `GET /api/auth/users/search?q=` (활성 사용자 검색)

### 6.2 ERP 동기화 플로우 (신규)

```
Admin 페이지 [ERP 동기화] 버튼
  → POST /api/auth/erp-sync
    → DataMart (DM_DATABASE_URL) me.emp_m 조회
    → 신규: INACTIVE 역할로 User 생성
    → 기존: erpId 연결 또는 name/dept/email 업데이트
    → 결과: { total, created, updated, skipped, errors }
```

### 6.3 사용자 비활성화 (신규)

- `INACTIVE` 역할 추가 — 로그인 차단 (403 응답)
- Admin 사용자 관리에서 역할 변경으로 접근 제어
- ERP 동기화 시 신규 사용자는 기본 `INACTIVE`로 생성 → Admin이 역할 승인
