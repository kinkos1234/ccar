## 1. 프로젝트 개요

Comad CAR(Customer Account Review) 시스템은  
VOC(Voice of Customer) 데이터를 통합 관리하고,  
AI 기반의 분석 및 전략 제언을 통해  
주간 리포팅 자동화까지 지원하는 내부 운영 플랫폼입니다.

기존에는 VOC 수집과 보고가 수기 기반으로 이루어져  
데이터 활용의 실효성이 낮고,  
전사적 피드백 분석 및 대응 체계에 한계가 있었습니다.

CAR 시스템은 이를 해소하고,  
각 지사의 VOC 데이터를 통합 관리하며,  
정기적인 전략 보고 프로세스를 표준화하고자 구축되었습니다.

## 2. 개발 목적

- 고객 VOC 데이터를 카테고리별로 수집, 스코어링, 누적 관리
- n8n + Ollama 기반 AI 분석 및 전략 제언 자동화를 통한 주간 보고 효율화
- 내부 협업 및 대응 체계의 가시성 강화
- Date Score 시스템을 통한 시간 관리 효율성 정량화
- Event Type별 점수 표시 분기화로 사용자 경험 최적화
- CAR 상세 모달 레이아웃 완전 통일로 일관성 확보
- 개선된 Dashboard 필터링 및 차트 상호작용을 통한 데이터 분석 효율성 증대
- 페이지네이션 및 테이블 정렬 기능으로 대용량 데이터 관리 효율성 향상
- 글로벌 법인 대응을 위한 완전한 다국어 지원 시스템 구축
- PostgreSQL 기반 운영 데이터베이스 (db.ccar.internal:5432)

## 3. 핵심 기능

### 3.1 기본 기능
- VOC 등록, 조회, 수정, 삭제 기능 (법인별 권한 분기)
- N:M 관계 기반 고객 담당자 관리
- JWT 기반 인증 및 4단계 권한 분리 (ADMIN/MANAGER/STAFF/INACTIVE)
- ERP(DataMart) 사용자 동기화 및 접근 권한 관리 (v2.4.3)
- N:M 내부 담당자 관리 (CarInternalContact, v2.4.3)

### 3.2 AI 분석 기능 (n8n + Ollama)
- n8n Workflow Automation + Ollama 기반 전략 제언 자동 생성 (OpenAI 의존성 제거)
- 고객사별 이슈 요약 및 상위 5개 이슈 추출
- Evidence(근거) 기반 전략 제언 생성
- Date Score 통합 분석으로 시간 관리 패턴 분석

### 3.3 보고서 및 메일 기능 (n8n + Comad 메일서버)
- n8n Workflow를 통한 주간 보고서 메일 자동 발송 (Comad 메일서버 mail.ccar.internal 경유, SendGrid 의존성 제거)
- n8n Schedule Trigger 기반 스케줄링 (node-cron 의존성 제거)
- 비동기 보고서 생성 및 진행 상황 실시간 추적
- 관리자 메일 주소 관리 시스템

### 3.4 대시보드 시각화
- 누적 VOC 데이터 기반 대시보드 시각화 (개선된 기간 필터 및 교차 필터링 포함)
- 페이지네이션 및 테이블 정렬 기능으로 사용성 대폭 향상
- 테이블 주제 컬럼 추가로 내용 직관성 개선
- 차트 상호작용 및 필터링 (PieChart, BarChart, LineChart)
- 고객사별 전략 제언 팝업 및 복사 기능
- Date Score 기반 지연 분석 및 시각화

### 3.5 Date Score 시스템
- CONTINUOUS 이벤트 기한 준수도 정량화: -5점~+5점 스케일
- 실시간 점수 계산: 백엔드에서 자동 계산 후 프론트엔드 표시
- Event Type별 UI 분기화: ONE_TIME(1열), CONTINUOUS(3열), 기타(4열)
- 모든 화면 적용: Dashboard 모달, /car 상세 모달, /car/[id] 페이지 완전 통일
- 점수 정합성 보장: 343개 CAR 데이터 점수 재계산 완료

### 3.6 Risk Mitigation
- riskMitigation (Boolean) - 리스크 완화 여부
- riskDescription (String?) - 리스크 상세 설명
- riskLevel (String?, default "MEDIUM") - LOW/MEDIUM/HIGH/CRITICAL
- Risk Analytics API (`/api/car/risk-analytics`)
- Risk Dashboard: Gantt 차트 기반 스코어보드 (법인별/카테고리별 매트릭스)
- RiskCheckbox, RiskIndicator 컴포넌트

### 3.7 Kafka 이벤트 로깅
- Kafka 기반 사용자 행동 추적 시스템
- 로그인/로그아웃, CUD 작업 이벤트 기록
- 토픽: system-user-events
- 환경변수로 활성화/비활성화 (KAFKA_ENABLED)

### 3.8 ERP 사용자 연동 및 접근 권한 관리 (v2.4.3)
- DataMart(dwp.ccar.internal:35432) 기반 사원 정보 조회 및 일괄 동기화 (`POST /api/auth/erp-sync`)
- ERP 동기화 시 신규 사원은 INACTIVE 역할로 자동 생성 → Admin이 역할 승인
- INACTIVE 역할: 로그인 차단, 내부 연락처 검색 제외
- User.erpId 필드를 통한 ERP 사원 ID 연동
- OTP 2FA 지원
- Admin 페이지에서 ERP 동기화 버튼 제공

### 3.9 내부 담당자 관리 (v2.4.3)
- CarInternalContact 중간 테이블을 통한 CAR ↔ User N:M 관계
- InternalContactSelector 컴포넌트 (타입어헤드 다중 선택)
- 활성 사용자 검색 API (`GET /api/auth/users/search`)

## 8. Date Score 시스템 가이드

### 8.1 Date Score 계산 규칙
- **적용 대상**: CONTINUOUS 이벤트만 (ONE_TIME 제외)
- **계산 기준**: 기한일(dueDate) vs 완료일(completionDate) 비교
- **점수 체계**:
  - 기한 내 완료: +5점
  - 1일 지연: +3점
  - 2~3일 지연: +2점
  - 4~7일 지연: 0점
  - 8~30일 지연: -3점
  - 30일 초과: -5점

### 8.2 Event Type별 점수 표시 분기화
- **ONE_TIME**: 주관적 점수만 표시 (1열 그리드, max-w-xs 제약)
- **CONTINUOUS**: 내부 점수, 고객 점수, Date Score (3열 그리드)
- **기타 Event Type**: 모든 점수 필드 표시 (4열 그리드)

### 8.3 UI 레이아웃 완전 통일
- **적용 범위**: Dashboard 테이블 클릭 모달, /car 상세 모달, /car/[id] 페이지
- **통일 요소**: 
  - 기본 정보 그리드 (2열)
  - 텍스트 섹션 (주제, Open Issue, Follow-up Plan)
  - 점수 상세 섹션 (Event Type별 분기)
  - 버튼 섹션 (닫기, 수정, 삭제)

### 8.4 점수 정합성 보장
- **전체 데이터 재계산**: 343개 CAR 데이터의 점수 수정 완료
- **실시간 동기화**: 백엔드 계산과 프론트엔드 표시 완전 일치
- **오류 방지**: BigInt, null, undefined 등 모든 엣지 케이스 처리

## 9. 대시보드 상호작용 정책

### 9.1 기본 상호작용 정책
- 필터 바는 대시보드 내 모든 구성 요소(KPI 카드, 차트, 테이블)에 전역적으로 영향을 미침
- 특정 차트 항목 클릭 시, 해당 항목은 필터에 **추가 적용**되며 UI 전체가 갱신됨
- 차트 항목을 다시 클릭하면 해당 항목만 필터에서 **제거**되며, 나머지 기존 필터는 그대로 유지됨
- KPI 카드 및 테이블 리스트도 동일한 필터 상태를 유지하며 데이터 필터링됨
- 필터 상태는 URL query로 반영되어 공유 및 재접속 시 동일한 뷰 유지 가능

### 9.2 개선된 기간 필터 정책
- **기본값**: 최근 6개월 (이전 30일에서 변경)
- **시작월만 설정**: 시작월부터 현재월까지 데이터 표시
- **종료월만 설정**: 데이터 최초월부터 종료월까지 표시  
- **둘 다 없음**: 데이터 전체 범위 표시 (최초월 ~ 최종월)
- 필터 초기화 시 데이터 범위 기반 적응형 기본값 설정

### 9.3 교차 필터링 및 데이터 일관성
- **교차 필터링**: 고객사 필터 선택 시 담당자 차트에서 해당 고객사 담당자만 표시
- **일관된 데이터 소스**: 프론트엔드 필터링된 데이터 우선 사용으로 모든 차트 동기화
- **실시간 반영**: 필터 변경 시 모든 UI 요소(차트, 테이블, KPI 카드)가 동시 업데이트

### 9.4 차트별 개선사항
- **LineChart**: X축 라벨을 "2024년 1월" 형식으로 년도 포함 표시
- **BarChart**: 활성 필터와 일치하는 항목에 색상 강조 및 테두리 표시
- **PieChart**: 필터링된 데이터 기준 실시간 상태 분포 업데이트

## 10. 로그인 페이지 정책

- 로그인은 `/login` 경로에서 이루어지며, ID/PW 입력을 통해 JWT 토큰을 발급받음
- 입력 필드는 `email`, `password`로 구성되며, 유효성 검사 실패 시 inline 에러 메시지 노출
- 로그인 실패 시 `toast`로 오류 알림이 표시됨 (`아이디 또는 비밀번호를 확인해주세요.`)
- 성공 시 `/dashboard`로 리디렉션됨
- JWT는 클라이언트 sessionStorage에 저장됨 (이전 localStorage에서 변경)
- 이후 모든 API 호출 시 `Authorization` 헤더로 전송됨
- UI는 중앙 정렬된 다크모드 폼 구조이며, 사용자는 로그인 외 별도 기능 접근 불가
