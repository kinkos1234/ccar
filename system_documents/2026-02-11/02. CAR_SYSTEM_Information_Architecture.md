```
Request → Cache Check → DB Query → Calculate → Cache Store → Response
    ↓          ↓           ↓          ↓           ↓           ↓
  Client    Memory      PostgreSQL    Backend     Memory     Client
             ↑                                   ↓
         Cache Hit                          Cache Miss
```

## 8. 글로벌화(i18n) 아키텍처

### 8.1 언어 처리 구조

```text
[사용자 선택 언어]
     ↓ (NEXT_LOCALE cookie 또는 쿼리 파라미터)
[i18n Context (Next.js App Router)]
     ↓
[언어별 JSON 리소스 로딩]
     ↓
[UI 텍스트 다국어 적용]
```
- 사용자가 선택한 언어는 NEXT_LOCALE cookie 또는 쿼리 스트링 ?lang=xx로 저장
- app/[lang]/page.tsx 형식으로 i18n 라우팅 분기 가능

### 8.2 언어별 리소스 구조
```pgsql
/locales
  ├─ ko/
  │   └─ common.json
  ├─ en/
  │   └─ common.json
  ├─ zh/
  ├─ vi/
  ├─ hi/
  └─ es-mx/
```
- 각 JSON은 동일 키 구조 유지
- 번역 누락 시 fallback 언어(en) 적용

### 8.3 글로벌 대응 대상
| 적용 대상     | 범위                         |
| --------- | -------------------------- |
| UI 요소     | 버튼, 메뉴, 필터, 라벨, 툴팁 등       |

| 모달 및 상세   | CAR 상세 모달, Date Score 설명 등 |

## 9. Risk Mitigation 아키텍처

### 9.1 데이터 모델
CAR 모델에 다음 Risk 관련 필드 추가:
- `riskMitigation` (Boolean) — Risk 해당 여부
- `riskDescription` (String?) — Risk 상세 설명
- `riskLevel` (String? @default("MEDIUM")) — Risk 수준 (LOW / MEDIUM / HIGH / CRITICAL)

### 9.2 API
- `/api/car/risk-analytics` 엔드포인트로 리스크 분석 데이터 제공
- 카테고리별, 고객사별, 법인별 리스크 통계 집계

### 9.3 프론트엔드 컴포넌트
- **RiskCheckbox**: CAR 등록/수정 시 Risk 여부 체크
- **RiskIndicator**: CAR 조회/표시 시 Risk 상태 표시
- **Risk Dashboard** (`/dashboard/risk`): Gantt 차트 기반 스코어보드

### 9.4 Risk Dashboard
- Gantt 차트 기반 스코어보드 (법인별/카테고리별 매트릭스)
- 카테고리별, 고객사별 리스크 통계 집계
- 리스크 수준별 필터링 및 시각화
