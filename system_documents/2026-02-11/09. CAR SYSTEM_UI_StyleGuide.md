## 1. UI 디자인 개요

### 1.1 디자인 철학
- **일관성**: 모든 페이지에서 동일한 디자인 패턴 사용
- **접근성**: 다양한 사용자와 환경에서 사용 가능한 인터페이스
- **효율성**: 최소한의 클릭으로 원하는 기능에 접근
- **국제화**: 다국어 환경에서 최적화된 사용자 경험

## 2. 다국어 지원 UI 가이드라인 (2025-01-04 신규 추가)

### 2.1 텍스트 길이 대응 디자인
```css
/* 긴 언어(베트남어, 스페인어)에 대한 UI 보호 */

  @apply flex-1 min-w-0; /* 최소 너비 보장 */

  @apply max-w-xs truncate; /* 긴 텍스트 잘림 처리 */

  @apply hidden;

```

### 2.2 언어 전환 컴포넌트 디자인
```jsx
const LanguageSwitcher = () => (
  <Select>
    <SelectTrigger className="w-[120px]">
      <Globe className="h-4 w-4" />

    </SelectTrigger>
    <SelectContent>
      {languages.map(lang => (
        <SelectItem key={lang.code} value={lang.code}>
          <span className="flag-icon">{lang.flag}</span>

```

### 2.3 다국어 반응형 레이아웃
- **최소 너비 보장**: 모든 언어에서 UI 요소가 최소 사용 가능한 크기 유지
- **유연한 그리드**: 텍스트 길이에 따라 자동 조정되는 레이아웃
- **우선순위 기반 표시**: 중요한 정보 우선, 부가 정보 조건부 표시

### 2.4 언어별 UI 최적화 전략
- **한국어/중국어**: 한자 기반 압축된 텍스트 → 여유 공간 활용
- **영어**: 표준 기준 언어 → 기본 레이아웃
- **베트남어/스페인어**: 긴 텍스트 → 공간 최적화 및 잘림 처리
- **힌디어**: 복잡한 문자 → 폰트 렌더링 최적화

## 5. 정보 계층 구조 정의 (2025-07-02)

### 5.1 정보 우선순위
1. **1순위**: 기본 식별 정보 (법인, 고객사, 상태)
2. **2순위**: 시간 정보 (발행일, 기한일, 완료일)
3. **3순위**: 점수 정보 (Event Type별 분기)
4. **4순위**: 상세 텍스트 (주제, 이슈, 계획)

### 5.2 시각적 위계
- **헤더 (h2)**: 20px, font-semibold, text-white
- **섹션 라벨**: 14px, font-medium, text-gray-300
- **데이터 값**: 16px, font-normal, text-white
- **보조 정보**: 12px, font-normal, text-gray-400

## 6. 다국어(i18n) 대응 UI 가이드

### 6.1 텍스트 길이 유연성

- 언어에 따라 텍스트 길이가 최대 2.5배까지 길어질 수 있음 (예: 영어 → 스페인어)
- 모든 버튼, 라벨, 카드 컴포넌트는 **`min-width`, `max-w-fit`**, **`truncate`, `break-words`** 조합 적용
- KPI 카드, 테이블 헤더, 모달 제목 등은 텍스트 길이 초과 시 **`line-clamp`** 또는 툴팁 적용

### 6.2 폰트 및 언어별 렌더링 최적화

| 언어 | 기본 폰트 | 대체 폰트 |
|------|-----------|-----------|
| ko   | Noto Sans KR | sans-serif |
| en   | Inter, Roboto | sans-serif |
| zh   | Noto Sans SC | Microsoft YaHei |
| vi   | Noto Sans | sans-serif |
| hi   | Noto Sans Devanagari | sans-serif |
| es-mx| Inter | sans-serif |

- Tailwind 기준 `font-family` 클래스에 `font-sans` 사용 권장
- 한자, 힌디어 등은 OS별 fallback 고려 필요

### 5.3 정렬 및 공간 대응 정책

- 모든 텍스트 요소는 `text-left` + `justify-start` 기준
- RTL 언어는 현재 지원하지 않음 (추후 도입 고려)
- 드롭다운, 테이블 셀, 팝업은 언어별 너비 차이 고려한 `px-3~6` 여백 설정

### 5.4 언어 드롭다운 컴포넌트

- 위치: 헤더 우측 끝 `🌐 아이콘 + 언어코드` 형태
- 컴포넌트 예시:
  ```tsx
  <DropdownMenu>
    <DropdownMenuTrigger><GlobeIcon className="mr-1" /> KR</DropdownMenuTrigger>
    <DropdownMenuContent>
      <DropdownMenuItem onClick={() => setLang('ko')}>한국어</DropdownMenuItem>
      <DropdownMenuItem onClick={() => setLang('en')}>English</DropdownMenuItem>
      <DropdownMenuItem onClick={() => setLang('zh')}>中文</DropdownMenuItem>
      ...
    </DropdownMenuContent>
  </DropdownMenu>

## 8. 접근성 가이드라인

### 8.1 키보드 네비게이션
- 모든 인터랙티브 요소는 키보드로 접근 가능
- Tab 순서는 논리적 흐름을 따름
- Enter/Space로 버튼 활성화
- Escape로 모달 닫기

### 8.2 색상 대비
- 텍스트 대비율 최소 4.5:1 (WCAG AA 기준)
- 중요한 정보는 색상에만 의존하지 않음
- Date Score는 색상과 함께 숫자 값 표시
