## 1. 최신 분석/제언 파이프라인 기반 프롬프트 설계

### 1.1 전체 구조
- **전체 CAR 이력(고객사별) → 서버 전처리(evidence: 반복/장기/패턴 요약) → 최근 30일+미종결 CAR 상세 → n8n AI Agent (Ollama)가 요약 및 전략 제언 생성 → DB 저장/활용**
- **v2.4.3 변경사항**: OpenAI (GPT-3.5/GPT-4o) 완전 제거. AI 분석은 n8n 워크플로우 내 AI Agent 노드 + Ollama 로컬 모델로 전환됨. 리포트 생성용 프롬프트(system prompt, user prompt)는 n8n 워크플로우에서 관리되며, CAR 시스템 코드에서는 더 이상 관리하지 않음.

### 1.2 프롬프트 설계 원칙
- 프롬프트에는 반드시 `[전체 이력 기반 요약]`(evidence)와 `[최근 30일+미종결 상세]`가 모두 포함되어야 함
- evidence(근거)는 서버에서 전처리된 반복/장기/패턴 요약/통계 등
- 상세 이슈는 최근 30일 신규+미종결 CAR의 상세 목록
- 토큰 한계/비용 최적화 위해 evidence는 압축, 상세 이슈는 최대 10개 항목까지 우선순위(score + importance 기준 정렬)로 전송됩니다.

### 1.2.1 다국어 지원 프롬프트 전략 (2025-01-04 완성)

#### 구현 완료 현황
- **UI 다국어화**: 6개 언어 완전 구현됨 (ko, en, es-mx, hi, vi, zh)
- **번역 시스템**: next-intl 기반 완전 구축
- **언어 전환**: LanguageSwitcher 컴포넌트로 실시간 전환 지원
- **UX 최적화**: 긴 언어에서 UI 깨짐 방지 완료

```
다국어 지원 기능을 구현할 때 다음 사항을 고려해주세요:

1. **지원 언어**: 한국어(ko), 영어(en), 스페인어-멕시코(es-mx), 힌디어(hi), 베트남어(vi), 중국어(zh)
2. **번역 시스템**: next-intl 기반 구현
3. **번역 키 구조**: 계층적 구조로 체계적 관리
4. **UI 최적화**: 언어별 텍스트 길이 차이 대응
5. **UX 고려사항**: 긴 언어에서 UI 깨짐 방지

구현 시 useTranslations hook을 사용하고, 모든 하드코딩된 텍스트는 번역 키로 대체해주세요.

```
번역 키를 생성할 때 다음 규칙을 따라주세요:

2. **계층 구조**: 대분류.소분류.항목 형태

예시:
- navigation.dashboard → "대시보드"
- common.save → "저장"
- car.eventTypes.CONTINUOUS → "지속적"

다국어 환경에서 UI 최적화 시 다음을 고려해주세요:

### 1.3 프롬프트 예시 (n8n AI Agent 워크플로우에서 관리)

> **v2.4.3 변경**: 아래 프롬프트는 더 이상 CAR 시스템 코드에서 관리하지 않습니다. n8n 워크플로우의 AI Agent 노드에서 system prompt와 user prompt로 정의되어 관리됩니다. 아래는 참고용 프롬프트 구조입니다.

- **n8n AI Agent System Prompt (CAR 분석용):**
  ```
  당신은 Comad CAR 시스템의 고객 품질 이슈 분석 전문가입니다.
  고객사별 CAR 이력 데이터를 기반으로 이슈 요약, 전략 제언, 예방 조치를 생성합니다.
  응답은 한국어로, 명확하고 실질적인 조언 위주로 작성합니다.
  각 항목은 2~3문장 이내로 간결하게 작성합니다.
  ```
- **n8n AI Agent User Prompt Template:**
  ```
  아래는 고객사 {{customerName}}의 전체 이력 요약(근거)와 최근 30일+미종결 이슈 요약입니다.
  1. 지금까지의 주요 문제 경향을 요약해 주세요.
  2. 현재 오픈 이슈에 대해 우선적으로 어떤 대응이 필요할지 전략적으로 제언해 주세요.
  3. 향후 유사 이슈 예방을 위한 조치 방향을 제시해 주세요.
  (각 항목은 2~3문장 이내, 명확하고 실질적인 조언 위주)
  ---
  [근거 요약]
  {{evidence}}
  [이슈 요약]
  {{summaryText}}
  ```
- **n8n 워크플로우 프롬프트 관리 정책:**
  - system prompt: AI Agent 노드의 "System Message" 필드에 정의
  - user prompt: AI Agent 노드의 "User Message" 필드에 템플릿으로 정의
  - 변수 바인딩: `{{customerName}}`, `{{evidence}}`, `{{summaryText}}` 등 n8n 표현식 사용
  - 프롬프트 수정 시 n8n 워크플로우 편집기에서 직접 수정 (CAR 코드 변경 불필요)

### 1.4 파싱/출력 정책
- evidence, summary, topIssues, aiRecommendation 등 구조화된 데이터로 파싱
- 전략 제언은 라벨 기반(예: [전략명]: ...), 5줄, 3줄 등 다양한 포맷에 구조적으로 대응
- 5줄 미만 응답은 누락 항목을 `'-'`로 채워 파싱 오류 방지 (예: 예상 효과 누락 시 자동 `'-'`)
- 모든 경로에서 evidence(근거)도 함께 출력/활용

### 1.5 예외/유연성
- LLM 응답 포맷 변화에도 evidence+상세 이슈 구조를 유지
- 프롬프트/파싱 예시, fallback 구조 명시

## 11. AI 연동 프롬프트 설계 및 데이터 전송 정책 (n8n AI Agent + Ollama)

> **v2.4.3 변경**: OpenAI (GPT-3.5/GPT-4o) 완전 제거. 모든 AI 분석은 n8n AI Agent 노드 + Ollama 로컬 모델로 수행됩니다. 리포트 생성 프롬프트는 n8n 워크플로우에서 관리됩니다.

### 11.1 프롬프트 설계 원칙
- n8n AI Agent (Ollama): 이슈 요약, 누적 이슈 분석, 오픈 이슈 대응 전략, 향후 대응 방향 제언을 단일 AI Agent가 처리
- 기존 GPT-3.5 요약 + GPT-4o 전략 제언의 2단계 파이프라인을 n8n AI Agent 단일 워크플로우로 통합
- 프롬프트는 n8n 워크플로우의 AI Agent 노드에서 system prompt / user prompt로 관리
- 프롬프트는 명확하고 실질적인 조언 위주로 작성

### 11.2 데이터 전송 포맷
- 고객사/담당자별 이슈 목록(배열) 및 주요 필드(open_issue, follow_up_plan, contact, score, importance, issueDate, dueDate, completionDate 등)
- contact 필드는 '이름직급' 형식, ','로 구분, 줄바꿈/공백 제거
- 전송 데이터는 1,500자 이내, 초과 시 자동 분할
- 예시:
```json
{
  "company": "ABC Corp",
  "issues": [
    {
      "open_issue": "납기 지연",
      "follow_up_plan": "다음 주 고객사 컨택 예정",
      "contact": "홍길동과장",
      "score": 2.3,
      "importance": 4,
      "issueDate": "2025-05-10",
      "dueDate": "2025-05-30",
      "completionDate": null
    }
  ]
}
```

### 11.3 프롬프트 예시 (n8n 워크플로우에서 관리)

- n8n AI Agent System Prompt:
  ```
  당신은 Comad CAR 시스템의 고객 품질 이슈 분석 전문가입니다.
  고객사별 CAR 이력 데이터를 기반으로 이슈 요약, 전략 제언, 예방 조치를 생성합니다.
  응답은 한국어로, 명확하고 실질적인 조언 위주로 작성합니다.
  각 항목은 2~3문장 이내로 간결하게 작성합니다.
  불필요한 반복/상투적 문구를 제거하고 1,500자 이내로 요약합니다.
  ```
- n8n AI Agent User Prompt Template:
  ```
  아래는 고객사 {{customerName}}의 누적 이슈 요약과 현재 오픈 이슈 목록입니다.
  1. 지금까지의 주요 문제 경향을 요약해 주세요.
  2. 현재 오픈 이슈에 대해 우선적으로 어떤 대응이 필요할지 전략적으로 제언해 주세요.
  3. 향후 유사 이슈 예방을 위한 조치 방향을 제시해 주세요.
  (각 항목은 2~3문장 이내, 명확하고 실질적인 조언 위주)
  ---
  [누적 이슈 요약]
  {{summaryText}}
  [현재 오픈 이슈]

  ```

### 11.4 정책 요약
- 프론트/백엔드는 데이터 준비/전송만 담당, 요약 및 전략 제언 생성은 n8n AI Agent (Ollama)가 전담
- n8n 워크플로우가 요약과 전략 제언을 단일 파이프라인으로 처리하며, 결과는 팝업/보고서 등 UI에 표시
- n8n 워크플로우 트리거(요약/전략적 제언 생성)는 매주 월요일 오전 8시 30분 자동 실행
- 실패/지연 시 예외처리 정책은 API 문서 참고

## 11.5 자동 실행 정책
- 요약 및 전략 제언 생성은 n8n 워크플로우의 Cron 트리거로 **매주 월요일 오전 8시 30분** 자동 실행
- n8n AI Agent (Ollama) 호출 실패/지연 시 재시도 및 예외처리 정책은 [API 문서] 참고
- 별도 API 명세 참조: [API 문서](./04. CAR_SYSTEM_API_Document.md)


## 12. 전략 제언 프롬프트
- 라벨 기반(예: [전략명]: ...)을 기본, LLM이 라벨 없이 5줄로 응답해도 파싱 가능하도록 설계
- 예시:
  [전략명]: 내부 커뮤니케이션 프로토콜 강화  
  [대상]: 고객 대응 담당자 전체  
  [요약]: 반복되는 응답 지연 문제 발생  
  [조치]: SLA 재정의 및 응답 가이드라인 배포  
  [예상 효과]: 고객 만족도 및 NPS 상승 기대
- 5줄: 전략명\n대상\n요약\n조치\n예상효과

### 12.1 CAR 분석 → 전략 제언 흐름

[CAR 전체 이력] --전처리--> [Evidence 요약] + [30일 상세 이슈]
→ n8n AI Agent (Ollama)가 요약 + 제언 통합 생성 → DB 저장 → 대시보드/메일 전송


## 13. Top 5 이슈 요약 프롬프트
- 번호/불필요한 설명 없이 한 줄 요약 리스트로 유도
- 규칙 기반 3개 미만이면 LLM 요약 fallback

## 14. 파싱 구조
- 라벨 기반 우선, 없으면 줄 순서대로 5개 항목 매핑
- 5줄 미만이면 '-'로 채움

## 15. 예외/유연성
- LLM 응답 포맷 변화에도 구조적으로 대응
- 프롬프트/파싱 예시, fallback 구조 명시

## 15.1 Fallback 예시
LLM이 아래처럼 잘못 응답한 경우:
```
- 전략 제안입니다:
  - 고객의 신뢰를 유지해야 합니다.
  - 문제를 빠르게 해결합시다.
```

→ 파싱 결과:
| 전략명                       | 대상 | 요약                       | 조치         | 예상 효과             |
|----------------------------|------|----------------------------|--------------|------------------------|
| 고객의 신뢰를 유지해야 합니다 | -    | 문제를 빠르게 해결합시다 | -            | -                      |

> ※ 예외 응답이 반복되거나 구조화 파싱이 어려울 경우, 응답 본문 전체를 raw 형태로 저장하고 분석 로그에 남깁니다. 추후 관리자 확인을 통해 패턴 개선 예정입니다.

- CONTINUOUS: 3열 그리드, 내부점수+고객점수+Date Score
- 기타 Event Type: 4열 그리드, 모든 점수 필드 표시
- Date Score 색상 코딩: 양수(green), 0(yellow), 음수(red)

### 16.3 통일된 모달 레이아웃 프롬프트
`generate unified car detail modal layout with consistent structure`

### 16.4 BigInt 직렬화 프롬프트
`generate bigint serialization utility for date timestamp fields`

조건:
