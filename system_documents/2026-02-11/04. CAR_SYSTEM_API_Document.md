## 2. 인증 및 사용자 관리 API ✅ **완전 구현됨**

- **목적**: 사용자 로그인 인증 및 JWT 발급
- **구현상태**: ✅ 완료 (다국어 오류 메시지 지원)
- **요청 형식**:
  ```json
  {
    "email": "user@example.com",
    "password": "securepassword"
  }
  ```
- **응답 형식 (성공)**:
  ```json
  {
    "token": "JWT-토큰값",
    "user": {
      "id": 1,
      "name": "김정현",
      "role": "MANAGER",
      "language": "ko"
    }
  }
  ```
- **다국어 지원**: 오류 메시지가 Accept-Language 헤더에 따라 적절한 언어로 반환

## 3. CAR 관리 API ✅ **완전 구현됨** + Date Score 통합

### GET `/api/car`
- **목적**: CAR 목록 조회 (필터링 지원)

### Date Score 계산 로직 (2025-07-02)
- **적용 조건**: `eventType === "CONTINUOUS"` AND `dueDate` AND `completionDate` 존재
- **계산 공식**:
  ```javascript
  function calcDateScore(dueDate, completionDate) {
    const diffDays = Math.floor((completionDate - dueDate) / (1000 * 60 * 60 * 24));
    if (diffDays <= 0) return 5;   // 기한 내 완료
    if (diffDays === 1) return 3;  // 1일 지연
    if (diffDays <= 3) return 2;   // 2~3일 지연
    if (diffDays <= 7) return 0;   // 4~7일 지연
    if (diffDays <= 30) return -3; // 8~30일 지연
    return -5;                      // 30일 초과
  }
  ```
- **null 처리**: `dueDate` 또는 `completionDate`가 없으면 `dateScore: null`
- **ONE_TIME 처리**: ONE_TIME 이벤트는 항상 `dateScore: null`

## 5. Date Score 관련 API 정책 (2025-07-02)

### 5.1 Date Score 응답 정책
- **CONTINUOUS 이벤트**: 모든 API에서 `dateScore` 필드 포함
- **ONE_TIME 이벤트**: `dateScore: null` 고정
- **기타 Event Type**: `dateScore` 계산 시도 후 조건에 따라 결정
- **데이터 타입**: `number | null` (소수점 첫째 자리까지)

### 5.2 Event Type별 API 응답 최적화
- **ONE_TIME**: `subjectiveScore` 중심, `internalScore`/`customerScore`/`dateScore` null 허용
- **CONTINUOUS**: `internalScore`, `customerScore`, `dateScore` 모두 활용
- **기타**: 모든 점수 필드 포함하되 조건부 계산

### 5.3 점수 계산 우선순위
1. **Date Score**: `dueDate`와 `completionDate` 기반 자동 계산
2. **Total Score**: Event Type에 따른 가중합 계산
3. **Sentiment Score**: 텍스트 분석 기반 계산 (별도 필드)

### 5.4 BigInt 직렬화 처리
- **일시 필드**: `issueDate`, `dueDate`, `completionDate`의 BigInt → Number 변환
- **안전한 변환**: BigInt 범위 초과 시 null 반환
- **클라이언트 호환**: 모든 타임스탬프가 JavaScript Number로 안전하게 직렬화

## 8-1. n8n 연동 API ✅ **신규 (v2.4.3)**

> n8n 워크플로우에서 호출하는 전용 API입니다. JWT 인증 대신 `X-N8N-Webhook-Secret` 헤더로 인증합니다.

### POST `/api/n8n/report/save`
- **목적**: n8n이 생성한 주간 보고서를 백엔드에 저장
- **구현상태**: ✅ 완료 (v2.4.3)
- **인증**: `X-N8N-Webhook-Secret` 헤더 (JWT 불필요)
- **요청 형식**:
  ```json
  {
    "weekStartDate": "2026-03-09",
    "weekEndDate": "2026-03-15",
    "summary": "주간 종합 요약 (n8n + AI 생성)",
    "topIssues": [...],
    "aiRecommendation": "AI 전략 제언",
    "evidence": "분석 근거 데이터"
  }
  ```
- **응답 형식 (성공)**:
  ```json
  {
    "success": true,
    "data": {
      "id": 789,
      "weekStartDate": "2026-03-09",
      "weekEndDate": "2026-03-15",
      "generatedAt": "2026-03-19T10:00:00.000Z"
    }
  }
  ```
- **인증 실패 (401)**:
  ```json
  {
    "error": "Invalid or missing X-N8N-Webhook-Secret"
  }
  ```

### GET `/api/n8n/health`
- **목적**: n8n에서 백엔드 상태 확인용 헬스체크
- **구현상태**: ✅ 완료 (v2.4.3)
- **인증**: `X-N8N-Webhook-Secret` 헤더 (JWT 불필요)
- **응답 형식**:
  ```json
  {
    "status": "ok",
    "timestamp": "2026-03-19T10:00:00.000Z"
  }
  ```

### n8n 인증 방식
- **헤더**: `X-N8N-Webhook-Secret: <shared-secret>`
- **환경변수**: `N8N_WEBHOOK_SECRET`에 설정된 값과 비교
- **JWT 불필요**: n8n 전용 API는 사용자 인증이 아닌 서비스 간 인증 사용

## 11. 보안 및 인증 정책 ✅ **완전 구현됨**

### JWT 토큰 정책
- **발급**: 로그인 성공 시 sessionStorage 저장 권장
- **유효기간**: 7일 (168시간, expiresIn: '7d')
- **갱신**: 자동 갱신 미지원, 재로그인 필요
- **검증**: 모든 보호된 API에서 Bearer 토큰 검증

### 권한 기반 접근 제어 (RBAC)
- **ADMIN**: 모든 API 접근 가능
- **MANAGER**: 사용자 관리 API 제외하고 접근 가능
- **STAFF**: 읽기 중심 + 본인 데이터만 수정/삭제 가능

### n8n 서비스 인증 (v2.4.3 신규)
- **대상**: `/api/n8n/*` 경로
- **인증 방식**: `X-N8N-Webhook-Secret` 헤더
- **JWT 불필요**: 서비스 간 통신용 shared secret 방식
- **환경변수**: `N8N_WEBHOOK_SECRET`

### 데이터 보안
- **민감 정보**: 비밀번호 bcrypt 해싱
- **SQL Injection**: Prisma ORM으로 방어
- **XSS**: 입력 데이터 검증 및 이스케이프
- **CORS**: 허용된 도메인만 접근 가능
