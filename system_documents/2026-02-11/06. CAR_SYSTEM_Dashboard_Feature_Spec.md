## 1. Date Score 시스템 통합

### 1.1 Date Score 개념
- **정의**: CONTINUOUS 이벤트의 기한 준수 정도를 정량화한 점수 (-5점 ~ +5점)
- **적용 대상**: `eventType === "CONTINUOUS"` AND `dueDate` AND `completionDate` 존재
- **표시 위치**: 모든 CAR 상세 모달, 테이블, 상세 페이지

### 1.2 Date Score 계산 규칙
```javascript
function calcDateScore(dueDate, completionDate) {
  const diffDays = Math.floor((completionDate - dueDate) / (1000 * 60 * 60 * 24));
  if (diffDays <= 0) return 5;   // 기한 내 완료: +5점
  if (diffDays === 1) return 3;  // 1일 지연: +3점
  if (diffDays <= 3) return 2;   // 2~3일 지연: +2점
  if (diffDays <= 7) return 0;   // 4~7일 지연: 0점
  if (diffDays <= 30) return -3; // 8~30일 지연: -3점
  return -5;                      // 30일 초과: -5점
}
```

### 1.3 Event Type별 점수 표시 분기화
- **ONE_TIME**: 주관적 점수만 표시 (1열 그리드, `max-w-xs` 제약)
- **CONTINUOUS**: 내부 점수, 고객 점수, Date Score (3열 그리드)
- **기타 Event Type**: 모든 점수 필드 표시 (4열 그리드)

### 1.4 UI 표시 정책
- **색상 코딩**: Date Score 값에 따른 색상 구분
  - `+5, +3`: 녹색 (성공)
  - `+2, 0`: 노란색 (주의)
  - `-3, -5`: 빨간색 (지연)
- **툴팁**: Date Score 클릭 시 계산 방식 설명 표시
- **null 처리**: 계산 불가 시 '-' 표시

## 3. 대시보드 구성요소

### 3.1 상단 필터바
- **Corporation 필터**: 법인별 데이터 필터링
- **Customer 필터**: 다중 선택 가능한 고객사 필터
- **Department 필터**: 부서별 필터링
- **기간 필터**: 최근 6개월 기본값, 적응형 범위 설정

### 3.2 KPI 카드 섹션
- **총 이벤트 수**: 전체 CAR 개수 표시
- **평균 점수**: 전체 CAR의 평균 점수
- **진행중 이벤트**: 미완료 CAR 개수
- **완료율**: 완료된 CAR 비율 표시
- **평균 Date Score**: CONTINUOUS 이벤트의 평균 Date Score 표시

### 3.3 차트 시각화 섹션
- **PieChart**: 상태별 분포 (진행중/완료/지연)
- **BarChart**: 법인/고객사/담당자별 누적 점수
- **LineChart**: 월별 추이 (년월 라벨링 적용)
- **Date Score 차트**: 시간 관리 효율성 시각화

### 3.4 데이터 테이블
- **CAR 목록**: 전체 CAR 데이터 표시
- **주제 컬럼**: mainCategory 기반 주제 표시 (파란색 배지 스타일)
- **Date Score 컬럼**: CONTINUOUS 이벤트의 Date Score 표시
- **페이지네이션**: 15개 항목/페이지
- **정렬 기능**: 날짜, 고객사, 부서, Score, Date Score 정렬
- **상세 모달**: 클릭 시 통일된 CAR 상세 정보 표시

### 3.5 다국어(i18n) 대응

#### 언어 설정 구조
- 네비게이션 우측 상단에 🌐 언어 선택 드롭다운 추가
- 선택된 언어는 `NEXT_LOCALE` cookie로 저장됨
- 모든 컴포넌트는 `useTranslation()` 훅 기반으로 언어 리렌더링 처리

#### 적용 범위
- KPI 카드 항목 제목: 총 이벤트 수, 평균 점수, 완료율 등
- 필터 항목 라벨: 법인, 고객사, 부서, 기간 등
- 테이블 컬럼 헤더 및 날짜 라벨
- 차트 툴팁, 축 라벨, 범례 등
- Date Score 색상 설명 툴팁 (예: "기한 내 완료" → 해당 언어로 표시)

#### 대응 언어
- 한국어 / 영어 / 중국어 / 베트남어 / 힌디어 / 멕시코 스페인어

#### 예외 처리
- 번역 누락 시 fallback 언어: 영어 (en)
- 테이블 정렬/페이지네이션 기능은 기존 로직 유지, 라벨만 번역

## 4. 필터링 시스템

### 4.1 기간 필터 정책 (완전 구현됨)

#### 기본 동작 규칙
- **기본값**: 최근 6개월 (접속일 기준 6개월 전 ~ 현재월)
- **시작월만 설정**: 시작월 ~ 현재월
- **종료월만 설정**: 데이터 최초월 ~ 종료월  
- **둘 다 없음**: 데이터 전체 범위 (최초월 ~ 최종월)
- **둘 다 설정**: 사용자 지정 범위

#### 기간 필터 초기화 로직
- 필터 삭제 시 `dataDateRange`가 있으면 해당 범위 사용
- `dataDateRange`가 없으면 최근 6개월 기본값 사용
- 모든 차트와 테이블이 동일한 기간 필터 적용

### 4.2 일반 필터 정책

#### 드롭다운 필터
- **법인 필터**: 실제 입력 데이터 기반 드롭다운
- **고객사 필터**: 다중 선택 지원
- **부서 필터**: 단일 선택
- **상태 필터**: 진행 상태별 필터링
- **Event Type 필터**: ONE_TIME/CONTINUOUS 분류

#### 필터 상호작용
- **전체 옵션**: 모든 필터에 '전체' 옵션 포함
- **URL 상태 유지**: 필터 상태가 URL에 반영되어 공유/재방문 시 동일 뷰
- **실시간 적용**: 필터 변경 시 모든 UI 요소 즉시 업데이트

### 4.3 차트 상호작용 상세 정책 (완전 구현됨)

#### 교차 필터링
- **고객사↔담당자**: 고객사 필터 선택 시 담당자 차트에서 해당 고객사의 담당자만 표시
- **일관된 데이터 소스**: 프론트엔드 필터링된 데이터를 우선 사용하여 모든 차트가 동기화
- **실시간 반영**: 필터 변경 시 모든 UI 요소(차트, 테이블, KPI 카드)가 동시 업데이트

#### 차트 클릭 동작
- **추가 필터링**: 차트 항목 클릭 시 해당 항목이 필터에 추가 적용
- **토글 기능**: 재클릭 시 해당 필터만 제거, 나머지 필터는 유지
- **다중 선택**: 여러 항목 병렬 선택 가능
- **모든 차트 동기화**: 클릭 시 모든 데이터가 동기화되어 갱신

## 5. Date Score 시각화 및 분석

### 5.1 Date Score 차트 신설
```typescript
// Date Score 히스토그램
<BarChart 
  data={dateScoreDistribution}
  title="Date Score 분포"
  xAxis="점수 구간"
  yAxis="건수"
  colors={['#ef4444', '#f59e0b', '#10b981']} // 빨강/노랑/초록
/>
```

### 5.2 지연 분석 테이블
- **Score Worst 5**: 전체 점수 기준 최악 5건 표시 (완료된 케이스만 대상, status === 'CLOSED')
- **지연 Worst 5**: 지연 일수 기준 최악 5건 표시 (진행중인 케이스만 대상, status !== 'CLOSED')
- **카테고리별 지연 현황**: mainCategory별 평균 지연 일수

### 5.3 Score Worst 5 필터링 로직 (2025-07-02 업데이트)
```javascript
// Dashboard의 Score Worst 5 필터링 로직
const worstScore = [...filteredCarsWithStatus]
  .filter(car => car.status === 'CLOSED') // 완료된 케이스만 대상
  .filter(car => (car.calculatedScore !== undefined && car.calculatedScore !== null) || (car.score !== undefined && car.score !== null))
  .sort((a, b) => (a.calculatedScore || a.score || 0) - (b.calculatedScore || b.score || 0))
  .slice(0, 5);
```

#### 필터링 기준
- **완료된 케이스만 대상**: `status === 'CLOSED'` 조건 적용
- **점수 보유 케이스**: `calculatedScore` 또는 `score` 값이 존재하는 케이스만 대상
- **전체 점수 기준**: Date Score가 아닌 전체 점수(`calculatedScore` 또는 `score`)로 정렬
- **비즈니스 로직**: 완료된 케이스만 보여주는 것이 개선점 도출에 더 적합함

#### 지연 Worst 5와의 차이점
- **Score Worst 5**: 완료된 케이스(`status === 'CLOSED'`) 대상
- **지연 Worst 5**: 진행중인 케이스(`status !== 'CLOSED'`) 대상
- **목적**: 과거 성과 분석 vs 현재 진행 상황 모니터링

### 5.4 시간 관리 KPI 추가
- **평균 Date Score**: 전체 CONTINUOUS 이벤트의 평균 Date Score
- **지연 비율**: 기한을 넘긴 이벤트 비율
- **평균 지연 일수**: 지연된 이벤트의 평균 지연 일수

- **UI 최적화**: 
  - 15개 이하일 때는 페이지네이션 자동 숨김
  - 필터링과 완전 연동되어 필터된 결과에 대해서만 페이지네이션 적용
  - 반응형 디자인으로 모바일/데스크톱 완벽 대응
- **상태 관리**: 필터 변경 시 자동으로 첫 페이지로 이동

### 7.3 테이블 정렬 기능 (완전 구현)
- **정렬 가능 컬럼**: 날짜, 고객사, 부서, Score, Date Score
- **3단계 토글**: 미정렬 → 오름차순 → 내림차순 → 미정렬
- **시각적 표시**: 
  - 정렬 중인 컬럼: 파란색 화살표 아이콘
  - 미정렬 컬럼: 회색 상하 화살표 아이콘
  - 헤더 hover 시 배경색 변화
- **정렬 로직**:
  - **날짜 필드**: YYYY-MM-DD 형식 자동 감지 후 시간순 정렬
  - **숫자 필드**: Score, Date Score 등 숫자 크기 순 정렬
  - **문자열 필드**: 한글-영문 가나다순 정렬 (한글 우선순위)
  - **null/undefined**: 안전 처리 (정렬 시 하단 배치)

### 7.4 정렬과 페이지네이션 연동 (완전 구현)
- **자동 페이지 이동**: 정렬 변경 시 자동으로 첫 페이지로 이동
- **3단계 조합**: 정렬된 전체 데이터에 대해 페이지네이션 적용
- **완전 연동**: 필터 + 정렬 + 페이지네이션 3단계 조합 완벽 지원

## 13. Risk Mitigation 대시보드 (v2.4.2 구현 확정)

### 13.1 Risk Dashboard 개요
- **경로**: /dashboard/risk
- **목적**: 수주 리스크 제거 활동 현황을 시각적으로 모니터링
- **구현 방식**: Gantt 차트 기반 스코어보드
- **데이터 소스**: riskMitigation === true인 CAR 데이터

### 13.2 스코어보드 테이블
- **카테고리 행**: Purchasing, Quality, Engineering, Developing
- **법인 열**: CMKR, CMMPL, CMVN, CMSJ, CMMX, CMCZ
- **통계**: 법인별/카테고리별 Total, Closed, In Progress, Delayed 건수
- **접이식 구조**: 법인별 섹션 확장/축소 가능
- **완료율/지연율**: 법인별 completion rate, delay rate 표시

### 13.3 Gantt 차트 시각화
- **타임라인**: 현재 기준 ±6개월 범위
- **진행률 표시**:
  - CLOSED: 100% (초록색 바)
  - IN_PROGRESS: 50% (노란색 바)
  - DELAYED: 0% (빨간색 바)
- **막대 정보**: Main Category, Internal Contact, 진행률(%)
- **진행 필(fill)**: 경과 시간 대비 마감일까지의 진행 표시

### 13.4 데이터 필터링
- 부서 기반 카테고리 매칭 (정확/대소문자 무시/부분 일치)
- customerContacts[].department 필드 기준 필터링
- CAR 클릭 시 상세 모달 오픈
