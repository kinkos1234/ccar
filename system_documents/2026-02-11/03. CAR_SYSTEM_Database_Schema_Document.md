## 2. 완전 구현된 모델 정의 ✅

### 2.1 User 모델 (사용자 관리)

```ts
model User {
  id                Int      @id @default(autoincrement())
  loginId           String   @unique
  password          String
  role              Role
  name              String
  department        String
  email             String?
  weeklyReportEmail Boolean  @default(false)
  preferredLanguage String   @default("ko")      // 사용자 선호 언어
  timezone          String   @default("Asia/Seoul") // 사용자 시간대
  createdAt         DateTime @default(now())
  updatedAt         DateTime @updatedAt

  erpId             String?  @unique @db.VarChar(50) // ERP(DataMart) 사원 ID 연동 키
  cars              CAR[]    @relation("CreatedCars")
  carInternalContacts CarInternalContact[]
}
```

#### 주요 특징
- **고유 식별**: `loginId`를 통한 로그인 ID 관리
- **4단계 권한**: Role enum을 통한 ADMIN/MANAGER/STAFF/INACTIVE 분리
- **ERP 연동**: `erpId` 필드를 통한 DataMart 사원 정보 연동 (unique)
- **이메일 관리**: 선택적 이메일 필드 및 주간 보고서 수신 설정
- **글로벌화**: `preferredLanguage`, `timezone` 필드를 통한 사용자별 언어/시간대 설정
- **관계**: CAR 테이블과 1:N 관계 (생성자 추적), CarInternalContact 중간 테이블과 1:N 관계

### 2.2 CAR 모델 (핵심 VOC 데이터)

```ts
model CAR {
  id                  Int      @id @default(autoincrement())
  corporation         String   // 우리회사(Comad) 지사 구분

  eventType           EventType
  issueDate           BigInt   // timestamp로 저장
  dueDate             BigInt?  // timestamp로 저장
  importance          Float

  internalContact     String?
  receptionChannel    String?
  mainCategory        String?
  openIssue           String?
  followUpPlan        String?
  completionDate      BigInt?  // timestamp로 저장

  internalScore       Float?
  customerScore       Float?
  subjectiveScore     Float?
  score               Float?

  sentimentScore      Float?
  aiKeywords          String?

  createdAt           DateTime @default(now())
  updatedAt           DateTime @updatedAt

  createdBy           Int
  creator             User     @relation("CreatedCars", fields: [createdBy], references: [id])

  scoreHistories      ScoreHistory[]
  carCustomerContacts CarCustomerContact[]
  carInternalContacts CarInternalContact[]
}
```

#### 주요 특징
- **타임스탬프 저장**: 날짜 필드는 BigInt 타입으로 밀리초 단위 저장
- **유연한 스코어링**: 이벤트 타입에 따른 다양한 점수 체계
- **AI 분석 필드**: sentimentScore, aiKeywords 등 AI 분석 결과 저장
- **N:M 관계**: 고객 담당자(CarCustomerContact) 및 내부 담당자(CarInternalContact) 중간 테이블을 통한 다중 관계

### 2.4 CarCustomerContact 모델 (N:M 중간 테이블)

```ts
model CarCustomerContact {
  carId               Int
  customerContactId   Int
  CAR                 CAR      @relation(fields: [carId], references: [id])
  CustomerContact     CustomerContact @relation(fields: [customerContactId], references: [id])
  @@id([carId, customerContactId])
}
```

#### 주요 특징
- **복합 기본키**: carId + customerContactId 조합
- **자동 관계 관리**: Prisma에서 자동으로 관계 생성/삭제
- **데이터 무결성**: 외래키 제약을 통한 참조 무결성 보장

model CarInternalContact {

## 4. 데이터베이스 관계 설계 ✅ **완전 구현됨**

### 4.1 관계 다이어그램
```text
User (1) ──────── (N) CAR
  │                  │
  │                  │ (N)
  │                  │
  │       CarCustomerContact (중간 테이블)
  │                  │
  │                  │ (N)
  │                  │
  │       CustomerContact (1) ─┘
  │                  │
  │ (N)              │ (N)
  │                  │
  └── CarInternalContact (중간 테이블) ──┘   ← v2.4.3 신규

CAR (1) ──────── (N) ScoreHistory

WeeklyReport (독립적)
managementEmail (독립적)
CorporationSettings (독립적) - 법인별 다국어/시간대 설정
emp_m (독립적) - DataMart 연동 레거시 사원 정보
```

### 4.2 주요 관계 설명
- **User ↔ CAR**: 1:N (사용자가 여러 CAR 생성 가능)
- **CAR ↔ CustomerContact**: N:M (CarCustomerContact 중간 테이블, 한 CAR에 여러 고객 담당자)
- **CAR ↔ User (내부 담당자)**: N:M (CarInternalContact 중간 테이블, v2.4.3 신규)
- **CAR ↔ ScoreHistory**: 1:N (CAR의 점수 변경 이력 추적)
- **독립 테이블**: WeeklyReport, managementEmail, CorporationSettings, emp_m은 다른 테이블과 직접 관계 없음

## 11. 🎯 **NEW**: Date Score 계산 시스템 (2025-07-02)

### 11.1 Date Score 개념
- **정의**: CONTINUOUS 이벤트의 기한 준수 정도를 정량화한 점수 (-5점 ~ +5점)
- **계산 대상**: `eventType === "CONTINUOUS"` AND `dueDate` AND `completionDate`
- **저장 방식**: DB에 저장하지 않고 백엔드에서 실시간 계산

### 11.2 Date Score 계산 로직
```javascript
function calculateDateScore(dueDate, completionDate) {
  if (!dueDate || !completionDate) return null;
  
  const diffDays = Math.floor(
    (new Date(completionDate) - new Date(dueDate)) / (1000 * 60 * 60 * 24)
  );
  
  if (diffDays <= 0) return 5;   // 기한 내 완료: +5점
  if (diffDays === 1) return 3;  // 1일 지연: +3점
  if (diffDays <= 3) return 2;   // 2~3일 지연: +2점
  if (diffDays <= 7) return 0;   // 4~7일 지연: 0점
  if (diffDays <= 30) return -3; // 8~30일 지연: -3점
  return -5;                      // 30일 초과: -5점
}
```
